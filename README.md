# Glint

专为 AI agent（Claude Code / OpenCode / Codex 等）设计的终端阅读层。
不替代 AI，不替代终端——只做一件事：**让你更快读懂 AI 在做什么、做了什么。**

## 解决什么问题

AI coding agent 的输出是一坨平铺的文本流。你要在几百行输出里人肉翻找：改了哪个文件？报错了吗？测试过了没？

Glint 把这坨文本变成结构化的、可导航的阅读界面。

## 四个核心功能

### 语义折叠

AI 输出按语义分块渲染，不再是无差别的文本流。

| 块类型 | 内容 |
|---|---|
| `[说话]` | AI 的解释、计划 |
| `[写文件]` | 文件写入 + 文件名 |
| `[执行命令]` | shell 命令 + 结果 |
| `[结果]` | 最终输出、总结 |

每块默认折叠为一行摘要，展开看全文。`j/k` 导航，`Enter` 展开/折叠。

### 执行后摘要

每轮 AI 执行完毕，底部自动生成摘要卡：

- 改了哪些文件（新增 / 修改 / 删除）
- 有无 error / warning（数量 + 类型）
- 测试执行结果

摘要卡在下一轮 prompt 前始终可见，支持一键跳转到对应改动块。

### Error Surfacing

实时扫描输出流，识别 error / warning / exception。

- 错误行高亮（不同级别不同颜色）
- 状态栏实时显示：`⚠ 2 warnings · ✗ 1 error`
- 出现 error 时侧边标记，不依赖滚屏发现

识别范围：stderr、常见关键词（error / exception / traceback / failed / fatal 等）。

### Prompt → Output 锚点

每次用户输入 prompt，作为锚点插入输出流。

- Session 变成 `prompt1 → outputs → prompt2 → outputs` 的可导航结构
- 侧边栏列出所有 prompt，点击跳转
- 支持给 prompt 打标签（"尝试1"、"auth重构" 等）
- Session 结束后可导出为 markdown

## 界面概念

```
┌─────────────────────────────────────┐
│ [项目名]  [session时长]  [⚠2 errors] │  ← 状态栏
├────┬────────────────────────────────┤
│    │ > prompt 1                     │  ← 锚点
│ P  │   ▶ [说话] 我来帮你实现...      │  ← 折叠块
│ R  │   ▶ [写文件] src/auth.py       │
│ O  │   ▶ [执行命令] pytest          │
│ M  │   ── 摘要卡 ──────────────────  │
│ P  │   改动: 3文件  ⚠1 warning      │
│ T  │                                │
│ S  │ > prompt 2                     │
│    │   ▶ [说话] 根据错误...          │
├────┴────────────────────────────────┤
│ $ █                                 │  ← 输入区
└─────────────────────────────────────┘
```

## 不做什么

- AI 能力（不接 API，不做补全，不做建议）
- 代码编辑
- 内置 git UI
- 多人协作
- 云同步

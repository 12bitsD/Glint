# Glint

AI Agent 时代的终端阅读器。

## 为什么做这个

用 Claude Code / OpenCode 写代码时，终端体验很原始：AI 一轮操作下来输出几百行，你得自己滚屏翻找——改了哪个文件？报错了没？测试跑过了吗？

终端本身的交互形态没跟上 agent 的工作方式。Glint 就是补这一层：把 AI 的输出流变成可折叠、可导航、可速览的结构。

不替代 AI，不替代终端。只加一个阅读层。

## 核心功能

**语义折叠** — AI 输出按语义分块（说话 / 写文件 / 执行命令 / 结果），每块折叠为一行摘要，`j/k` 导航，`Enter` 展开。不用再在纯文本里找信息。

**执行后摘要** — 每轮结束自动生成摘要卡：改了几个文件、有无报错、测试结果。下一轮 prompt 前始终可见，支持跳转到对应改动。

**Error Surfacing** — 实时扫描输出流，error / warning 自动高亮 + 状态栏计数 + 侧边标记。不用等滚完才发现出错了。

**Prompt 锚点** — 每条 prompt 作为锚点，session 变成 `prompt → output → prompt → output` 的可导航结构。侧边栏列出所有 prompt，可打标签，可导出 markdown。

## 界面概念

```
┌─────────────────────────────────────┐
│ [项目名]  [session时长]  [⚠2 errors] │
├────┬────────────────────────────────┤
│    │ > prompt 1                     │
│ P  │   ▶ [说话] 我来帮你实现...      │
│ R  │   ▶ [写文件] src/auth.py       │
│ O  │   ▶ [执行命令] pytest          │
│ M  │   ── 摘要卡 ──────────────────  │
│ P  │   改动: 3文件  ⚠1 warning      │
│ T  │                                │
│ S  │ > prompt 2                     │
│    │   ▶ [说话] 根据错误...          │
├────┴────────────────────────────────┤
│ $ █                                 │
└─────────────────────────────────────┘
```

## 不做什么

不接 AI API，不做代码编辑，不做 git UI，不做协作，不做云同步。
